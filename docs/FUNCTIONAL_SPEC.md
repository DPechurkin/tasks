# Функциональная спецификация — Momentum

---

## 1. Общие принципы

- Приложение работает полностью офлайн — все ресурсы (JS, CSS, шрифты) загружаются локально
- Один пользователь, без авторизации
- Тёмная тема Bootstrap 5.3 на всём приложении
- Навигация между тремя разделами через боковую панель или верхнюю навигацию
- Все списки без постраничной навигации — единый вертикальный скролл

---

## 2. Раздел "Идеи"

### 2.1 Список идей

- Отображается как единый скроллируемый список карточек
- Карточки упорядочены по полю `order` — вверху самые важные, внизу наименее важные
- Каждая карточка отображает:
  - Заголовок идеи
  - Первые ~150 символов описания (с многоточием при обрезке)
  - Количество привязанных планов (бейдж)
  - Дата создания
  - Иконки действий: редактировать, удалить

### 2.2 Приоритизация через drag-and-drop

- Любую карточку можно зажать и перетащить на новое место
- После отпускания порядок сохраняется в БД немедленно
- Визуальный placeholder показывает место вставки во время перетаскивания

### 2.3 Создание новой идеи

- Кнопка "Новая идея" присутствует:
  - В верхней части списка (добавляет перед первой карточкой)
  - После каждой карточки (добавляет сразу после неё)
- При нажатии открывается модальное окно с формой:
  - Поле "Название" (обязательное, макс. 200 символов)
  - Поле "Описание" (textarea, необязательное, без ограничений)
  - Кнопки: "Сохранить" / "Отмена"
- Новая карточка вставляется в выбранную позицию

### 2.4 Редактирование идеи

- По клику на иконку редактирования открывается то же модальное окно с заполненными полями
- После сохранения карточка обновляется без перезагрузки страницы

### 2.5 Удаление идеи

- По клику на иконку удаления появляется диалог подтверждения
- При удалении идеи привязанные планы НЕ удаляются — они становятся "без идеи"

### 2.6 Просмотр идеи (детальная страница)

- Клик по заголовку или телу карточки открывает детальную страницу идеи
- На детальной странице отображается:
  - Полный заголовок и описание
  - Секция "Привязанные планы" — список ссылок на планы
    - Каждая ссылка показывает: название плана, краткое описание
    - Клик по ссылке — переход на страницу плана
  - Кнопка "Привязать план" — открывает выпадающий список всех существующих планов (с поиском)
  - Возможность отвязать план (иконка ✕ рядом с планом в списке)

## 3. Раздел "Планы"

### 3.1 Список планов

- Единый скроллируемый список карточек
- Все планы отображаются вне зависимости от привязки к идее
- Карточки упорядочены по полю `order` (перетаскиваемые для приоритизации)
- Каждая карточка отображает:
  - Название плана
  - Краткое описание (~100 символов)
  - Если план привязан к идее: бейдж с названием идеи (кликабельный — ведёт на идею)
  - Количество задач в плане (бейдж)
  - Дата создания

### 3.2 Создание нового плана

- Кнопка "Новый план" вверху списка и после каждой карточки
- Форма:
  - Название (обязательное)
  - Краткое описание (необязательное)
  - Привязка к идее (необязательная, выпадающий список идей с поиском)
- Новый план вставляется в выбранную позицию

### 3.3 Редактирование и удаление плана

- Редактирование через модальное окно (аналогично идеям)
- При удалении плана появляется предупреждение: "Будут удалены все задачи плана и их расписание. Продолжить?"
- Удаление каскадное: план → задачи → слоты расписания

### 3.4 Детальная страница плана

Открывается по клику на карточку плана. Содержит:

#### 3.4.1 Шапка плана
- Название и полное описание
- Если привязан к идее: ссылка "← Идея: [название идеи]"
- Кнопки: редактировать план, удалить план

#### 3.4.2 Список задач
- Задачи отображаются как карточки в упорядоченном списке
- Порядок задач определяет их последовательность выполнения
- Задачи перетаскиваются для изменения порядка
- Каждая карточка задачи отображает:
  - Порядковый номер (автоматически по позиции в списке)
  - Название задачи
  - Первые ~100 символов описания
  - Статус задачи (цветной бейдж)
  - Количество запланированных слотов в расписании (иконка календаря + число)
  - Количество комментариев (иконка + число)
  - Кнопки: редактировать, удалить

#### 3.4.3 Создание задачи
- Кнопка "Новая задача" вверху списка и после каждой карточки
- Форма:
  - Название (обязательное)
  - Описание (textarea, необязательное)
  - Статус (по умолчанию "Новая")

#### 3.4.4 Детальная страница задачи
- Открывается по клику на карточку задачи
- Содержит:
  - Название и полное описание
  - Текущий статус (с возможностью изменить прямо на странице)
  - Ссылка "← План: [название плана]"
  - Секция "Расписание задачи" — список всех слотов (дата, время, комментарий)
    - Каждый слот: дата, время с/по, комментарий (если есть)
    - Слоты отсортированы по дате (от новых к старым)
  - Кнопка "Поставить в расписание" — открывает форму добавления слота

## 4. Раздел "Расписание"

### 4.1 Общий макет

Экран разделён на две области:
- **Левая панель** (~35% ширины): три сетки календарей
- **Правая панель** (~65% ширины): детали выбранного дня + лента задач

### 4.2 Левая панель — Навигация и календари

#### 4.2.1 Навигатор месяца
- Отображается над тремя сетками
- Показывает название "текущего выбранного месяца" (по умолчанию — реальный текущий месяц)
- Кнопки "←" (предыдущий) и "→" (следующий) для переключения
- При переключении три сетки обновляются:
  - Верхняя: месяц до выбранного
  - Средняя: выбранный месяц
  - Нижняя: месяц после выбранного

#### 4.2.2 Сетка месяца
- Стандартная сетка: 7 колонок (Пн-Вс), строки — недели
- Каждая ячейка дня содержит:
  - Цифру дня
  - Число запланированных задач (маленький бейдж, если > 0)
- Текущая реальная дата: выделена цветом (accent)
- Прошедшие даты: приглушённый цвет
- Выбранная дата: обводка/подсветка
- При клике на день — загружается правая панель

#### 4.2.3 Метки месяцев
- Над каждой сеткой подпись: название месяца и год
- Метка текущего реального месяца дополнительно отмечена

### 4.3 Правая панель — Детали дня и лента

#### 4.3.1 Заголовок
- Отображает выбранную дату: "Воскресенье, 6 апреля 2025"
- Если выбранная дата — сегодня: дополнительная метка "Сегодня"
- Если дата в прошлом: метка "Прошедший день" и иконка замка

#### 4.3.2 Задачи выбранного дня
Список слотов расписания для выбранного дня:

**Для будущей даты (включая сегодня от текущего времени):**
- Каждый слот отображает: время с/по, название задачи, название плана, статус задачи
- Кнопка удаления слота
- Кнопка редактирования времени слота
- Кнопка "Добавить задачу в этот день" — открывает форму:
  - Выбор плана (выпадающий список)
  - Выбор задачи из плана (выпадающий список, фильтрованный по плану)
  - Время начала (time picker)
  - Время окончания (time picker)
  - Валидация: новый слот не должен пересекаться с уже существующими слотами этого дня

**Для прошедшей даты:**
- Список слотов только для просмотра
- Нет кнопок добавить/удалить/изменить время
- Рядом с каждым слотом: поле для комментария (textarea, кнопка сохранить)
- Если комментарий уже есть — показывается с возможностью редактирования

#### 4.3.3 Лента будущих задач
- Под задачами выбранного дня — непрерывный список задач, начиная со следующего дня после выбранного
- Задачи сгруппированы по датам
- Каждая группа: дата-заголовок + список слотов этой даты
- Слот в ленте показывает: время с/по, название задачи, план
- Лента скроллируется вниз
- Максимальная глубина ленты: 90 дней вперёд от выбранной даты

### 4.4 Правила редактирования дат

| Условие | Разрешено |
|---|---|
| Дата в будущем | Добавлять, изменять, удалять слоты |
| Сегодняшняя дата, время слота ещё не наступило | Добавлять, изменять, удалять слоты |
| Сегодняшняя дата, время слота уже прошло | Только комментарий |
| Дата в прошлом | Только комментарий к существующим слотам |

### 4.5 Добавление задачи в расписание из раздела "Планы"

- На детальной странице задачи кнопка "Поставить в расписание"
- Форма: выбор даты (date picker), время с/по
- Валидация пересечений: запрос к API проверяет свободность слота
- После добавления — ссылка "Посмотреть в расписании" ведёт на раздел расписания с открытой нужной датой

### 4.6 Валидация пересечений слотов

- Нельзя добавить слот, если в тот же день у другой задачи есть слот который пересекается по времени
- Правило: `new_start < existing_end AND new_end > existing_start`
- При конфликте: сообщение об ошибке с указанием конкретного конфликтующего слота

## 5. Статусы задач

| Код | Название | Цвет бейджа |
|---|---|---|
| `new` | Новая | secondary (серый) |
| `in_progress` | Что-то делалось | warning (жёлтый) |
| `done` | Успешно сделана | success (зелёный) |
| `done_partially` | Сделана, но не совсем так | info (голубой) |
| `abandoned` | Не сделана и не будет делаться | danger (красный) |

Статус задачи:
- Меняется на детальной странице задачи (dropdown или набор кнопок-статусов)
- Меняется прямо на карточке задачи в списке (быстрый dropdown)
- Виден во всех местах где отображается задача (карточка, слот расписания)

## 6. Уведомления

### 6.1 Браузерные push-уведомления (Service Worker)

- При первом открытии приложения: запрос разрешения на уведомления
- Service Worker регистрируется и работает в фоне
- Уведомление отправляется за 15 минут до начала задачи
- Уведомление содержит: название задачи, название плана, время начала
- Клик по уведомлению открывает приложение на соответствующей дате

### 6.2 In-page уведомления

- Если приложение открыто в браузере: Toast-уведомление (Bootstrap Toast) за 5 минут до задачи
- Toast содержит: название задачи, оставшееся время
- Toast не исчезает автоматически — требует ручного закрытия или нажатия "Открыть"

### 6.3 Технический механизм

- Service Worker опрашивает API `/api/notifications/upcoming?minutes=20` каждые 5 минут
- API возвращает слоты расписания, начинающиеся в ближайшие `minutes` минут
- Service Worker сравнивает с уже отправленными уведомлениями (хранит ID в IndexedDB) чтобы не дублировать

## 7. Связи между разделами

### 7.1 Идея → Планы
- На детальной странице идеи: список привязанных планов (ссылки)
- Клик — переход на детальную страницу плана
- Привязка/отвязка плана от идеи

### 7.2 План → Идея
- На детальной странице и карточке плана: ссылка на идею (если привязан)
- Изменение привязки через редактирование плана

### 7.3 Задача → Расписание
- Со страницы задачи: кнопка "Поставить в расписание" + список всех слотов
- Из расписания: клик на задачу в слоте → переход на страницу задачи

### 7.4 Расписание → Задача → План → Идея
- Цепочка навигации всегда доступна через breadcrumb или обратные ссылки

## 8. Навигация приложения

### 8.1 Структура маршрутов

```
/                       → Редирект на /ideas
/ideas                  → Список идей
/ideas/:id              → Детальная страница идеи
/plans                  → Список планов
/plans/:id              → Детальная страница плана
/plans/:planId/tasks/:taskId → Детальная страница задачи
/schedule               → Раздел расписания (текущая дата)
/schedule/:date         → Расписание с открытой конкретной датой
```

### 8.2 Боковая/верхняя навигация

- Всегда видимая навигационная панель
- Три пункта: Идеи / Планы / Расписание
- Активный раздел подсвечен

---

## 9. Общие требования к UX

- Все операции создания/редактирования через модальные окна (без перехода на отдельную страницу формы)
- После каждого сохранения — список обновляется без полной перезагрузки страницы
- Все деструктивные действия (удаление) требуют подтверждения
- При загрузке данных — Bootstrap spinner
- При ошибке API — Toast с сообщением об ошибке
- Все формы валидируются на клиенте и на сервере
