# User Cases — Momentum

**Актор:** Пользователь (единственный, без авторизации)

---

## Раздел "Идеи"

### UC-001 Просмотр списка идей
**Предусловие:** Пользователь открывает приложение или переходит в раздел "Идеи"
**Шаги:**
1. Переходит на `/ideas`
2. Видит список карточек идей, упорядоченных по приоритету (вверху важнейшие)
3. Каждая карточка показывает: название, фрагмент описания, количество привязанных планов, дату создания

**Ожидаемый результат:** Список загружен, карточки видны в правильном порядке
**Граничные случаи:** Список пустой → отображается пустое состояние с кнопкой создать первую идею

---

### UC-002 Создание идеи в начале списка
**Предусловие:** Пользователь находится на `/ideas`
**Шаги:**
1. Нажимает кнопку "Новая идея" над первой карточкой (или над пустым списком)
2. Открывается модальное окно
3. Вводит название (обязательно) и описание (опционально)
4. Нажимает "Сохранить"

**Ожидаемый результат:** Новая карточка появляется первой в списке, страница не перезагружается
**Граничные случаи:**
- Пустое название → форма не отправляется, поле подсвечено красным
- Название > 200 символов → ошибка валидации

---

### UC-003 Создание идеи после конкретной карточки
**Предусловие:** В списке есть как минимум одна идея
**Шаги:**
1. Наводит курсор на карточку → появляется кнопка "Новая идея" под ней
2. Нажимает кнопку
3. Заполняет форму и сохраняет

**Ожидаемый результат:** Новая идея вставляется сразу после выбранной карточки, сохраняя правильный order

---

### UC-004 Редактирование идеи
**Предусловие:** В списке есть идея
**Шаги:**
1. Нажимает иконку редактирования (✎) на карточке
2. Открывается модальное окно с заполненными текущими значениями
3. Изменяет название и/или описание
4. Нажимает "Сохранить"

**Ожидаемый результат:** Карточка обновляется немедленно, позиция в списке не меняется

---

### UC-005 Удаление идеи
**Предусловие:** Идея привязана к 2 планам
**Шаги:**
1. Нажимает иконку удаления (🗑) на карточке
2. Появляется диалог подтверждения
3. Нажимает "Удалить"

**Ожидаемый результат:** Идея удалена, карточка исчезает из списка. Привязанные планы остаются, но их `idea_id` обнулён (бейдж идеи исчезает с их карточек)
**Граничные случаи:** Нажатие "Отмена" → ничего не меняется

---

### UC-006 Приоритизация идей через drag-and-drop
**Предусловие:** В списке 3+ идеи
**Шаги:**
1. Зажимает drag handle (⠿) на карточке
2. Перетаскивает вверх или вниз
3. Отпускает на новой позиции

**Ожидаемый результат:** Карточка встаёт на новое место немедленно (оптимистично). Порядок сохраняется в БД. После перезагрузки страницы порядок сохранён.
**Граничные случаи:** Ошибка сети при сохранении → карточка возвращается на исходную позицию, Toast с ошибкой

---

### UC-007 Просмотр детальной страницы идеи
**Предусловие:** Идея привязана к 2 планам
**Шаги:**
1. Нажимает на название идеи в карточке
2. Переходит на `/ideas/:id`

**Ожидаемый результат:** Отображается полное описание идеи и список привязанных планов в виде кликабельных ссылок

---

### UC-008 Привязка плана к идее
**Предусловие:** Существует идея и минимум один план без этой идеи
**Шаги:**
1. На странице идеи нажимает "Привязать план"
2. Появляется dropdown с поиском по всем существующим планам
3. Выбирает нужный план

**Ожидаемый результат:** План появляется в списке привязанных планов на странице идеи. В карточке этого плана появляется бейдж с названием идеи. В плане появляется обратная ссылка на идею.
**Граничные случаи:** Попытка привязать уже привязанный план → план не дублируется в списке

---

### UC-009 Отвязка плана от идеи
**Предусловие:** К идее привязан план
**Шаги:**
1. На странице идеи нажимает ✕ рядом с планом в списке
2. Подтверждает (или без подтверждения — действие не деструктивное)

**Ожидаемый результат:** План исчезает из списка привязанных планов. В плане исчезает ссылка на идею. Сам план и его задачи не удаляются.

---

### UC-010 Переход из идеи к плану
**Предусловие:** К идее привязан план
**Шаги:**
1. На странице идеи нажимает на название плана в списке

**Ожидаемый результат:** Переход на `/plans/:id` детальную страницу плана

2. В поле "Идея" выбирает из dropdown нужную идею
3. Сохраняет

**Ожидаемый результат:** Новый план создан с бейджем идеи. На странице идеи этот план появляется в списке привязанных планов.

### UC-015 Каскадное удаление плана
**Предусловие:** План содержит 3 задачи, у двух задач есть слоты в расписании
**Шаги:**
1. На карточке плана нажимает удаление
2. Видит предупреждение: "Будут удалены все задачи (3) и их записи в расписании (N). Продолжить?"
3. Подтверждает удаление

**Ожидаемый результат:** Удалены план + 3 задачи + все слоты расписания этих задач. Бейдж задач в расписании обновлён.

---

### UC-016 Просмотр задач плана
**Предусловие:** План содержит задачи
**Шаги:**
1. Нажимает на карточку плана
2. Переходит на `/plans/:id`

**Ожидаемый результат:** Список задач с порядковыми номерами, статусами, счётчиками слотов и комментариев

---

### UC-017 Изменение статуса задачи из карточки
**Предусловие:** Задача в статусе "Новая"
**Шаги:**
1. На карточке задачи нажимает на бейдж статуса
2. Появляется dropdown со всеми 5 статусами
3. Выбирает "Что-то делалось"

**Ожидаемый результат:** Бейдж немедленно меняется на жёлтый "Что-то делалось"

4. Сохраняет

**Ожидаемый результат:** Задача создана, вставлена в нужное место, порядковые номера остальных задач пересчитаны

### UC-019 Переупорядочивание задач (изменение последовательности)
**Предусловие:** В плане 4 задачи
**Шаги:**
1. Перетаскивает задачу #3 на место #1

**Ожидаемый результат:** Задача встаёт первой, остальные сдвигаются, порядковые номера обновляются. Порядок сохраняется в БД.

### UC-020 Просмотр детальной страницы задачи
**Предусловие:** Задача имеет 2 слота в расписании с комментариями
**Шаги:**
1. Нажимает на карточку задачи

**Ожидаемый результат:**
- Полное описание задачи
- Breadcrumb: Идея → План → Задача (если идея привязана)
- Текущий статус с возможностью изменить
- Список всех слотов расписания: дата, время, комментарий (если есть), отсортированы от новых к старым

---

### UC-021 Смена статуса задачи из детальной страницы
**Шаги:**
1. На `/plans/:planId/tasks/:taskId` нажимает dropdown статуса
2. Выбирает новый статус

**Ожидаемый результат:** Статус обновляется, badge меняет цвет, `updatedAt` обновляется

---

### UC-022 Постановка задачи в расписание со страницы задачи
**Предусловие:** Пользователь на странице задачи
**Шаги:**
1. Нажимает "Поставить в расписание"
2. Открывается модальное окно
3. Выбирает дату (08.04.2025), время 10:00–12:00
4. Сохраняет

**Ожидаемый результат:**
- Слот создан
- В списке слотов задачи появляется новая запись
- В ячейке календаря на 08.04 бейдж увеличивается на 1
- Появляется ссылка "Открыть в расписании"

**Граничные случаи:**
- Выбранное время пересекается с уже существующим слотом → ошибка 409 с указанием конфликта (задача + время)
- Конечное время ≤ начальному → ошибка валидации

## Раздел "Расписание"

### UC-023 Открытие расписания
**Шаги:**
1. Переходит на `/schedule`

**Ожидаемый результат:**
- Левая панель: три сетки — прошлый месяц (сверху), текущий (в центре), следующий (снизу)
- Текущая дата подсвечена в сетке
- Навигатор месяцев показывает текущий месяц
- Правая панель: ни один день не выбран → пустое состояние или подсказка

---

### UC-024 Навигация по месяцам
**Предусловие:** Отображается апрель 2025 (текущий)
**Шаги:**
1. Нажимает "→" (следующий месяц)

**Ожидаемый результат:**
- Навигатор показывает "Май 2025"
- Верхняя сетка: апрель 2025
- Средняя сетка: май 2025
- Нижняя сетка: июнь 2025
- При следующем "→": июнь/июль/август и т.д.

**Граничные случаи:** Переключение на 6 месяцев вперёд — все сетки корректно отображают правильные месяцы и годы

---

### UC-025 Выбор будущей даты
**Предусловие:** В расписании на 10 апреля запланирована 1 задача
**Шаги:**
1. Нажимает на ячейку "10" в сетке апреля

**Ожидаемый результат:**
- Ячейка "10" получает выделение
- Правая панель: заголовок "Четверг, 10 апреля 2025"
- Карточки задач с временными слотами
- Кнопка "Добавить задачу в этот день" активна
- Лента: задачи с 11 апреля и далее

---

### UC-026 Добавление задачи на будущую дату из расписания
**Предусловие:** Выбрана будущая дата, существует план с задачами
**Шаги:**
1. Нажимает "Добавить задачу в этот день"
2. В инлайн-форме выбирает план "Исследование рынка"
3. Автоматически загружается список задач плана
4. Выбирает задачу "Анализ конкурентов"
5. Устанавливает время: 14:00 – 16:00
6. Нажимает "Добавить"

**Ожидаемый результат:**
- Слот создан и сразу отображается в правой панели
- Бейдж в ячейке дня увеличивается на 1
- Форма сворачивается

## Навигация и связи

### UC-041 Полная цепочка навигации Идея → План → Задача
**Шаги:**
1. `/ideas` → нажимает на идею → `/ideas/:id`
2. Нажимает на привязанный план → `/plans/:id`
3. Нажимает на задачу → `/plans/:planId/tasks/:taskId`
4. На странице задачи видит breadcrumb: "Создать компанию > Исследование рынка > Анализ конкурентов"

**Ожидаемый результат:** Каждый переход работает, breadcrumb отражает правильные ссылки

---

### UC-042 Обратная навигация через breadcrumb
**Предусловие:** Пользователь на `/plans/:planId/tasks/:taskId`
**Шаги:**
1. Нажимает "Исследование рынка" в breadcrumb → переход на `/plans/:id`
2. Нажимает "Создать компанию" → переход на `/ideas/:id`

**Ожидаемый результат:** Каждая ссылка breadcrumb работает корректно
